OpenClaw 配置汇总：openclaw.json 完整指南

11 Jun 2026
# 4404 字 ~ 约 12 分钟
  • openclaw.json 是 OpenClaw 的核心配置文件，存放于 ~/.openclaw/openclaw.json，支持 JSON 和 JSON5 格式。所有字段均为可选，省略时使用安全默认值。本文按模块逐一汇总各配置项。

gateway — 网关配置

控制 Gateway 的监听、认证、热重载和推送等行为。

字段说明示例 / 默认值
gateway.port监听端口18789（默认）
gateway.bind绑定地址"127.0.0.1"
gateway.mode运行模式"local"
gateway.auth.token认证 Token字符串
gateway.auth.password认证密码字符串
gateway.auth.mode认证模式—
gateway.reload.mode热重载策略"hot" / "restart" / "hybrid"
gateway.reload.debounceMs防抖延迟（毫秒）300
gateway.handshakeTimeoutMs握手超时（毫秒）—
gateway.channelHealthCheckMinutes渠道健康检查间隔（分钟）—
gateway.push.apns.relayAPNs 推送中继地址—
gateway.controlUi.enabled是否启用控制台 UItrue
gateway.controlUi.root控制台 UI 静态资源目录"$HOME/.openclaw/control-ui-custom"
💡 端口优先级
CLI 参数 > 环境变量 > 配置文件 > 默认值 18789。

agents — 代理配置

定义 AI Agent 的默认参数、代理列表与路由绑定。

字段说明示例 / 默认值
agents.defaults.workspaceAgent 工作目录"~/.openclaw/workspace"
agents.defaults.model.primary默认主模型"anthropic/claude-opus-4-6"
agents.defaults.thinkingDefault默认思考深度"high"
agents.defaults.timeoutSeconds单次执行超时（秒）1800
agents.defaults.heartbeat.every心跳间隔，"0m" 表示禁用"0m"
agents.defaults.skills默认启用的技能列表—
agents.defaults.sandbox沙箱策略配置—
agents.list命名 Agent 列表（可覆盖 defaults）—
agents.bindingsAgent 路由绑定规则—

channels — 渠道配置

配置各聊天渠道（WhatsApp、Telegram 等）的接入策略。

字段说明示例 / 默认值
channels.whatsapp.allowFromWhatsApp 白名单号码["+15555550123"]
channels.<provider>.dmPolicy私聊策略—
channels.<provider>.groupPolicy群聊策略—
channels.<provider>.healthMonitor渠道健康监控配置—
ℹ️ 私信安全
每个渠道（whatsapp、telegram 等）均可独立配置 dmPolicy、groupPolicy 等字段。私信强制安全配对。

session — 会话配置

管理会话隔离、重置与线程绑定。

字段说明示例 / 默认值
session.dmScope / session.scope会话隔离粒度"per-sender"
session.resetTriggers / session.reset会话重置命令["/new", "/reset"]
session.threadBindings线程与 Agent 的绑定—

messages — 消息配置

控制消息可见性与群聊行为。

字段说明
messages.visibleReplies可见回复策略
messages.groupChat群聊消息处理规则

tools — 工具配置

管理 Agent 可调用的工具集合。

字段说明示例
tools.allow工具白名单—
tools.deny工具黑名单（优先于 allow）—
tools.profile工具预设配置集"coding"（含 FS/运行时等）
tools.elevated沙箱外提权控制—
tools.sessions.visibility会话工具可见范围"tree"（默认）
tools.media音视频处理模型配置—
tools.experimental.planTool实验性计划工具（需严格 agentic 模型）—

models — 模型配置

定义 AI 模型及自定义提供商。

字段说明
models.providers自定义模型提供商配置

结合 auth-profiles.json 管理 API Key 与认证档案。

mcp.servers — MCP 服务器配置

配置 MCP (Model Context Protocol) 服务器连接。

skills — 技能配置

管理 Agent 可用的技能插件。

plugins — 插件配置

扩展功能的插件注册与管理。

cron — 定时任务配置

字段说明
cron.enabled是否启用定时任务
cron.maxConcurrentRuns最大并发执行数
cron.runLog执行日志配置

hooks — 钩子配置

字段说明
hooks.enabled是否启用钩子
hooks.token钩子鉴权 Token
hooks.path钩子脚本路径
hooks.mappings事件到钩子的映射规则

env — 环境变量配置

字段说明
env.vars注入的自定义环境变量
env.shellEnv从 Shell 环境继承的变量列表

secrets — 密钥配置

通过 SecretRef 引用敏感信息，支持三种来源：

字段说明
source来源类型："env" / "file" / "exec"
provider密钥提供商
id密钥标识（env 类型需匹配 ^[A-Z][A-Z0-9_]{0,127}$）

auth.profiles — 认证档案

管理 API Key 和认证凭据。

JSON auth-profiles.json
{
  "version": 1,
  "profiles": {
    "openrouter:default": {
      "type": "api_key",
      "provider": "openrouter"
    }
  }
}

其他配置模块

以下模块在配置文件中同样存在，但通常无需手动修改：

模块说明
browser控制 Agent 的浏览器自动化能力
commitments管理 Agent 的承诺/约束行为
discoveryAgent 或服务的自动发现机制
ui控制台 UI 的定制化选项
logging日志级别与输出目标
diagnostics运行诊断与故障排查
identityAgent 的身份标识信息
update自动更新策略
acpAgent Communication Protocol 相关配置
cli命令行工具的偏好设置
wizard初始化向导相关选项

环境变量

除配置文件外，还可通过以下环境变量覆盖默认行为：

环境变量说明
OPENCLAW_HOMEOpenClaw 主目录
OPENCLAW_STATE_DIR状态目录
OPENCLAW_CONFIG_PATH配置文件路径

完整配置示例

JSON5 openclaw.json
{
  // 网关
  gateway: {
    mode: "local",
    port: 18789,
    reload: { mode: "hybrid", debounceMs: 300 },
    controlUi: {
      enabled: true,
      root: "$HOME/.openclaw/control-ui-custom"
    }
  },

  // 代理
  agents: {
    defaults: {
      workspace: "~/.openclaw/workspace",
      model: { primary: "anthropic/claude-opus-4-6" },
      thinkingDefault: "high",
      timeoutSeconds: 1800,
      heartbeat: { every: "0m" }
    }
  },

  // 渠道
  channels: {
    whatsapp: {
      allowFrom: ["+15555550123"]
    }
  },

  // 会话
  session: {
    scope: "per-sender",
    resetTriggers: ["/new", "/reset"]
  },

  // 工具
  tools: {
    profile: "coding",
    sessions: { visibility: "tree" }
  },

  // 定时任务
  cron: {
    enabled: true,
    maxConcurrentRuns: 3
  },

  // 环境变量
  env: {
    vars: {},
    shellEnv: []
  }
}
