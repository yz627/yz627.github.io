OpenClaw 目录结构详解

11 Jun 2026
# 2781 字 ~ 约 7 分钟

顶层概览

OpenClaw 的运行时数据存储在 ~/.openclaw/ 目录下，整体结构如下：

~/.openclaw/
├── agents/                  # Agent 运行时数据（会话、持久化状态）
│   └── Subagent-name/       # 额外 Agent 运行时数据
├── cache/                   # 缓存目录
├── completions/             # Shell 自动补全脚本
├── credentials/             # 凭证文件（飞书 Bot 授权等）
├── devices/                 # 已配对设备信息
├── exec-approvals.json      # 命令执行审批配置
├── extensions/              # 插件扩展（飞书等）
├── feishu/                  # 飞书去重状态
├── flows/                   # 工作流注册表（SQLite）
├── identity/                # 设备身份与密钥
├── memory/                  # Agent 记忆数据库
├── npm/                     # 插件 npm 包管理
├── openclaw.json            # 主配置文件
├── openclaw.json.bak*       # 配置备份（多个历史版本）
├── openclaw.json.last-good  # 上次正常启动的配置快照
├── plugins/                 # 插件安装状态
├── plugin-skills/           # 插件技能（软链接指向实际位置）
├── sandboxes/               # 沙箱 Agent 工作空间
├── state/                   # OpenClaw 主状态数据库
├── subagents/               # 子 Agent 运行记录
├── tasks/                   # 任务调度数据库
├── tools/                   # 工具链
├── tui/                     # TUI 最近会话记录
├── update-check.json        # 更新检查记录
└── workspace/               # Agent 工作空间（用户配置、文档）
    └── Subagent-name workspace/

核心目录详解

openclaw.json——主配置文件

这是整个 OpenClaw 的核心配置文件，定义了所有 Agent 的身份、模型、工作空间和消息路由规则。支持热加载，修改后无需重启即可生效。

agents/——Agent 运行时数据

每个 Agent 目录下包含两个子目录：

  • agent/ — Agent 持久化状态、记忆数据，其中 auth-profiles.json 存储模型认证数据，models.json 存储模型配置（所有 Agent 共用）
  • sessions/ — 会话记录、对话历史

workspace/——Agent 工作空间

每个 Agent 在 workspace 下有独立目录，包含以下标准文件：

文件用途
AGENTS.md子 Agent 调度配置（maxSpawnDepth、maxConcurrent）
SOUL.mdAgent 人格/行为定义
IDENTITY.mdAgent 身份标识
USER.md用户信息记录
TOOLS.md工具权限声明
HEARTBEAT.md心跳检查任务配置
BOOT.md启动检查清单
BOOTSTRAP.md首次运行仪式
memory/YYYY-MM-DD.md每日记忆日志
.openclaw/workspace-state.json工作空间状态

典型 Agent 目录结构：

  • director/ — 技术总监，SOUL.md 定义了飞书群调度流程
  • coding-agent/ — 编码专家，定义了代码实现能力
  • testing-agent/ — 测试工程师，定义了测试能力
  • product-agent/ — 产品经理，定义了需求分析能力

memory/——Agent 记忆

按日期记录的对话记忆文件（如 2026-06-08-1636.md、2026-06-08-2300.md），用于 Agent 在多次会话间保持上下文连续性。

flows/——工作流注册表

使用 SQLite 数据库存储已注册的工作流定义，与 Lobster YAML 工作流引擎配合使用。

subagents/——子 Agent 运行记录

记录所有 SubAgent 的运行历史，包括创建时间、任务描述、执行状态和结果摘要。

credentials/——凭证文件

存储各类服务的认证凭证，如飞书 Bot 授权信息等。

extensions/——插件扩展

存放已安装的插件扩展，如飞书集成等。

plugin-skills/——插件技能

通过软链接指向实际的插件技能目录，实现技能的动态加载和管理。

sandboxes/——沙箱工作空间

为需要隔离执行的 Agent 提供独立的沙箱环境。

state/——主状态数据库

OpenClaw 主进程的状态数据库，记录平台级别的运行状态。

tasks/——任务调度数据库

存储任务调度相关的数据，与 flows/ 配合实现工作流的调度和执行。

identity/——设备身份

存储设备身份信息和密钥对，用于设备间的认证和通信。

devices/——已配对设备

记录已配对的设备信息，支持多设备协同工作。

completions/——Shell 自动补全

存放 Shell 自动补全脚本，提升命令行使用体验。

exec-approvals.json——命令审批

定义哪些命令需要人工审批才能执行，实现安全控制。

配置文件版本管理

OpenClaw 对配置文件提供了多层次的版本保护：

文件用途
openclaw.json当前生效的主配置
openclaw.json.bak*配置备份（多个历史版本）
openclaw.json.last-good上次正常启动的配置快照
openclaw.json.pre-update更新前的配置快照
openclaw.json.clobbered*被覆盖的旧配置

这种设计确保了配置的安全性——即使配置文件被错误修改，也可以从备份中恢复。

总结

OpenClaw 的目录结构体现了其模块化的设计思想：

  • 配置与运行分离：openclaw.json 定义静态配置，agents/、sessions/ 存储运行时数据
  • 工作空间隔离：每个 Agent 拥有独立的 workspace/ 目录，包含人格定义、工具权限、记忆日志等
  • 插件化扩展：通过 extensions/、plugin-skills/、plugins/ 实现能力的动态扩展
  • 安全控制：credentials/、exec-approvals.json、sandboxes/ 多层次保障安全
  • 可观测性：memory/、subagents/、flows/ 提供完整的运行轨迹
