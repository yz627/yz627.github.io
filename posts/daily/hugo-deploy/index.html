Hugo 构建与部署

10 Jun 2026
# 950 字 ~ 约 3 分钟

本地构建

hugo            # 生成静态站点到 public/
hugo -D         # 包含草稿
hugo --minify   # 压缩 HTML/CSS/JS

本地预览

hugo server -D

默认监听 http://localhost:1313，文件变更时自动热重载。

部署到 GitHub Pages

1. 创建仓库

  • <username>.github.io（用户主页）
  • 或任意仓库 + 启用 Pages

2. 部署脚本

#!/bin/bash
hugo --minify
cd public
git init
git add .
git commit -m "Deploy $(date +%Y-%m-%d)"
git push -f git@github.com:<user>/<repo>.git main

3. GitHub Actions 自动化

name: Deploy Hugo
on:
  push:
    branches: [main]
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: peaceiris/actions-hugo@v3
        with:
          hugo-version: 'latest'
          extended: true
      - run: hugo --minify
      - uses: peaceiris/actions-gh-pages@v4
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./public

部署到 Cloudflare Pages

  1. 连接 GitHub 仓库
  2. 构建命令：hugo --minify
  3. 输出目录：public
  4. 环境变量：HUGO_VERSION=0.125.0

性能优化清单

项目措施
图片WebP 转换 + srcset
CSS/JS--minify 压缩
字体font-display: swap
资源CDN 加速

小结

Hugo 的部署是纯静态文件复制，几乎零成本，可搭配任意静态托管服务。
