AI 开发核心概念速览：从 Prompt 到 MCP

10 Jun 2026
# 2410 字 ~ 约 7 分钟

AI 开发领域涌现了大量新概念，很多术语看起来各自独立，实际上层层递进、互相关联。本文梳理六个最核心的概念，帮你建立一张完整的认知地图。

Prompt：与 AI 对话的起点

Prompt 是用户提供给 AI 模型的输入文本，用于描述任务、设定语境、约束输出格式或引导推理方向。它是人类意图与模型能力之间的翻译层。

大语言模型本质上是自回归的概率预测器——给定前文序列，计算下一个词的概率分布。Prompt 中的每个词都会通过 Transformer 的注意力机制与其他词交互，改变隐层表征，从而影响输出。

一个好的 Prompt 能将模型的注意力集中到训练数据中与任务相关的特定模式上。比如加上"让我们一步步思考"，模型的预测概率分布就会从"直接猜答案"切换为"复现解题过程"的模式。这就是所谓的"咒语"背后的数学逻辑。

Agent：从"会聊天"到"能做事"

Agent（智能体）是能够围绕目标自主完成任务的智能系统。

如果说大语言模型擅长"理解与生成"，Agent 则更进一步：它在 LLM 的基础上，结合任务规划、工具调用、记忆管理、环境反馈等能力，形成一个完整的"感知—决策—行动—迭代"闭环。

Agent 不是单纯的"会聊天的模型"，而是一个面向目标执行任务的系统形态。你给它一个目标，它会自己拆解步骤、调用工具、处理异常、迭代推进，直到任务完成。

RAG：给 AI 加上"查资料"的能力

RAG（Retrieval-Augmented Generation，检索增强生成）是一种将信息检索与大语言模型生成结合的系统架构。

核心思路很简单：在模型生成答案之前，先从外部知识库中检索出与当前问题最相关的文档片段，再把这些内容与用户问题一起提供给模型，让它基于检索到的资料生成更准确、更可控的回答。

可以把 RAG 理解为给 LLM 增加了一个"先查资料，再回答"的能力层。这解决了 LLM 的两个核心短板：知识过时和幻觉问题。模型不再只依赖训练时学到的知识，而是能实时引用最新的、特定领域的内容。

Function Calling：让 AI 调用外部工具

Function Calling 让大语言模型能以结构化方式请求调用外部函数或工具。

它并非让模型直接执行函数，而是让模型在需要时输出一个包含函数名和参数的标准化数据（如 JSON），由应用程序代码来实际执行。

这相当于定义了大模型调用工具的接口结构。有了 Function Calling，LLM 不再只是一个文本生成器，而是可以触发数据库查询、API 调用、文件操作等真实动作。Agent 的"工具调用"能力，底层往往就依赖 Function Calling 机制。

MCP：统一 AI 的连接标准

MCP（Model Context Protocol，模型上下文协议）是一种开放式协议，用来规范应用程序向 LLM 提供上下文与工具能力的方式。

可以把 MCP 想象为"AI 应用的 USB-C 接口"：统一连接方式，让模型能以标准化结构访问数据源与工具。

MCP 架构分为两端：

MCP Server（服务端） 是能力与上下文的提供方，对外暴露三类标准接口：

  • Tools：可被模型调用的操作，如查询数据库、读写文件、调用外部 API
  • Resources：结构化或非结构化的数据源，如文件、知识库、业务数据
  • Prompts：可复用的提示模板，帮助模型稳定完成特定任务

MCP Client（客户端） 是能力的使用方，通常嵌入在 AI 应用或 IDE 中，负责与 Server 建立连接、发起调用、将结果回填到上下文继续驱动模型。

工作流程：Client 连接 Server 获取能力清单 → 用户提出需求 → 模型判断需要哪些工具 → Client 按协议调用并拿到结果 → 结果回填上下文继续推理。

Skill：可复用的能力操作手册

Skill 是"可复用的能力模块"，通常包含目标、触发条件、执行步骤和边界约束。

可以把 Skill 理解为"教模型如何正确完成任务的一张卡片"。它比 Prompt 更强调可执行性和复用性，可以简化理解为 Prompt 加载器。

Skill 与 MCP 的关系很清晰：MCP 负责"连接与调用"的标准化（技能本身），Skill 负责"如何用好这些能力"的方法论与流程（操作手册）。

举个例子：一个"总结会议纪要"的 Skill 会定义目标是把录音转成结构化纪要，触发条件是用户说"帮我整理会议纪要"，步骤是获取音频→转写→提取决议和待办→输出模板，边界是缺少音频或权限时提示用户。

它们之间的关系

这六个概念不是孤立的，而是构成了一个层层递进的体系：

Prompt（单次交互）
  ↓ 结构化、可复用
Skill（操作手册）
  ↓ 驱动
Agent（自主执行系统）
  ↓ 通过
Function Calling（工具调用接口）
  ↓ 标准化为
MCP（统一连接协议）
  ↓ 增强知识
RAG（外部知识检索）

Prompt 是最基础的交互方式；Skill 把 Prompt 封装成可复用的模块；Agent 用 Skill 和规划能力驱动复杂任务；Function Calling 让 Agent 能调用外部工具；MCP 把工具连接标准化；RAG 则为整个系统补充外部知识。

理解了这条脉络，再看到新的 AI 概念时，就能快速定位它在整个体系中的位置。
